# 序列推荐模型项目说明

## 项目概述

本项目实现了一个基于Transformer架构的序列推荐模型，能够处理用户行为序列数据并预测用户下一个可能交互的物品。模型融合了多种类型的特征（稀疏特征、数组特征、多模态特征等），并通过注意力机制捕捉序列中的依赖关系，适用于个性化推荐场景。

## 核心功能

- 处理多类型用户和物品特征，包括稀疏特征、连续特征、数组特征和多模态嵌入特征
- 利用改进的Transformer架构捕捉用户行为序列中的时间和语义依赖
- 支持训练过程中的负采样和权重调整，优化模型性能
- 提供完整的训练和推理流程，支持推荐结果的评估

## 代码结构

```
项目主要根目录/
├── model.py        # 模型定义，包含网络结构和核心逻辑
├── dataset.py      # 数据集处理，包含数据加载、特征处理等
└── main.py         # 主程序，包含训练、推理和评估流程
```

## 模型架构

### 核心组件

1. **BaselineModel**：主模型类，继承自`torch.nn.Module`，包含模型的完整结构

2. **特征处理模块**：
   - 支持多种特征类型（稀疏特征、连续特征、数组特征、多模态特征）
   - 特征嵌入与转换，将不同类型特征映射到统一维度空间

3. **注意力机制**：
   - 基于FlashMultiHeadAttention实现高效注意力计算
   - 集成Rotary Position Embedding（RoPE）增强位置感知能力
   - 支持因果掩码（causal mask）确保序列顺序性

4. **归一化与前馈网络**：
   - 采用RMSNorm归一化层替代传统LayerNorm
   - 实现带残差连接的PointWiseFeedForward网络

### 模型工作流程

1. **特征处理**：将原始特征转换为模型可处理的张量形式，并生成对应的嵌入表示
2. **特征融合**：合并多类型特征，通过MoE（Mixture of Experts）结构进行特征整合
3. **序列编码**：添加位置编码（包括轴向位置编码），通过多层Transformer编码器处理序列
4. **预测输出**：根据编码后的序列表示，预测用户下一个可能交互的物品

## 数据集处理

### MyDataset类

数据集类负责数据加载、预处理和特征工程，主要功能包括：

1. **数据加载**：从预处理文件中加载用户序列数据，支持快速随机访问
2. **特征工程**：
   - 时间特征处理：将时间戳转换为星期、小时等稀疏时间特征
   - 时间差特征：计算序列中相邻行为的时间差并进行分桶处理
   - 缺失值填充：为缺失特征填充默认值
3. **负采样**：基于物品曝光次数的分层负采样策略，提升模型泛化能力
4. **数据增强**：生成序列特征、正样本特征和负样本特征，用于模型训练

### 特征类型

支持的特征类型包括：
- 用户稀疏特征：`103`, `104`, `105`, `109`
- 物品稀疏特征：`100`, `117`, `118`, `101`, `102`等
- 用户数组特征：`106`, `107`, `108`, `110`
- 多模态嵌入特征：根据配置的`mm_emb_ids`确定

## 使用方法

### 训练流程

1. 数据准备：确保数据文件按指定格式存放，包括序列数据、特征字典和索引文件
2. 参数配置：设置模型参数（隐藏层维度、注意力头数、Transformer块数等）和训练参数（学习率、批大小等）
3. 模型初始化：创建`BaselineModel`实例，并初始化权重
4. 训练模型：使用训练数据迭代更新模型参数，通过正负样本计算损失

### 推理流程

1. 加载模型：加载训练好的模型权重
2. 生成候选物品嵌入：预处理候选物品集，生成对应的嵌入向量
3. 序列编码：对用户行为序列进行编码，生成用户兴趣表示
4. 相似度计算：计算用户表示与候选物品表示的相似度，获取推荐结果
5. 评估指标：计算Recall@10和NDCG@10等指标评估推荐效果

## 关键技术亮点

1. **多特征融合**：灵活处理多种类型特征，通过专用网络层转换后进行有效融合
2. **高效注意力计算**：使用Flash Attention加速训练和推理过程，支持PyTorch 2.0+特性
3. **位置编码增强**：结合常规位置编码和轴向位置编码，更好地捕捉序列位置信息
4. **分层负采样**：基于物品曝光率的分层负采样策略，平衡正负样本分布
5. **动态权重调整**：在损失计算中增加转化行为的权重，提升推荐的实际效果

## 评估指标

模型训练和推理过程中主要关注以下指标：
- Recall@10：衡量模型在前10个推荐结果中命中用户实际交互物品的比例
- NDCG@10：衡量推荐结果的排序质量，考虑相关性和位置因素
- 综合得分：0.69*NDCG@10 + 0.31*Recall@10
